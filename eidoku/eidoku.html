Eidoku

A Neuro-Symbolic Verification Gate for LLM Reasoning

Eidoku is a lightweight System-2 verification gate for LLM outputs. It evaluates generated reasoning sequences using structural feasibility, not probability.

Hallucinations are treated as structural failures: statements that cannot be embedded into the contextual semantic structure without excessive deformation.

Core principle

Verification is formulated as a constraint satisfaction problem over semantic structure. Candidates exceeding a context-calibrated Violation Cost threshold are deterministically rejected.
  • Generation and verification are optimization-independent
  • Verification operates on feasibility, not likelihood
  • Rejection is explicit (no “least bad” fallback)
  • Thresholds are calibrated per context (not learned)

What Eidoku Checks

Each reasoning step is evaluated via an additive Violation Cost composed of:

  • Structural cost — graph connectivity (unsupported entity transitions)
  • Geometric cost — deviation from the local embedding manifold
  • Logical cost — entailment / contradiction via NLI

These proxies are statistically normalized and combined into:

J(S) = Σ τ(Sᵢ, Sⱼ)

A candidate is rejected if any local junction exceeds the calibrated critical threshold τc.

Architecture

System-1 (LLM) generates candidates
↓
Eidoku (System-2) computes Violation Cost
↓
Reject if ∃ step > τc
↓
Select lowest-cost accepted candidate
(or return NULL)

No gradients, no retraining, no decoding modification. Eidoku attaches to any LLM as a post-generation gate.

Claims / Non-Claims

  • Eidoku does NOT estimate truth
  • Eidoku does NOT improve generation
  • Eidoku enforces contextual structural closure
  • Eidoku rejects smooth falsehoods that probability cannot detect

If all candidates violate structure, Eidoku returns a null set. This is intentional.
